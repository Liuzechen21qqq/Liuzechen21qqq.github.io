OS绪论

计算机系统概述

程序：存放在磁盘中的可执行文件，包括指令、数据等。

任务：程序执行需要加载到内存中，并配备CPU及其他所需资源，执行中的程序便成了一个任务。

操作系统的概念

操作系统 (Operating System)
是指控制和管理整个计算机系统的硬件和软件资源，合理地组织、调度计算机的工作与资源地分配，进而为用户和其他软件提供方便接口的环境地程序集合。

OS是计算机系统中最基本地系统软件。

操作系统的功能与目标

  1. 操作系统是计算机系统资源的管理者
  2. 操作系统作为用户与计算机硬件系统之间的接口
  • 命令接口
    • 联机命令接口
    • 脱机命令接口
  • 程序接口：由一组系统调用组成
  1. 操作系统实现了对计算机资源的扩充

操作系统的特征

  1. 并发
  2. 共享
  3. 虚拟
  4. 异步

并发和共享是OS最基本的两个特征，二者互为存在条件。

操作系统发展历程

graph LR
    A[手工阶段] --> B[脱机处理]
    B --> C[早期批处理]
    C --> D[多道批处理]
    D --> E[分时操作系统]
    E --> F[实时操作系统]
  1. 手工阶段：独占计算机资源、资源利用率低；
  2. 脱机处理：减少了CPU的空闲时间，提高了I/O速度；
  3. 早期批处理；
  4. 多道批处理：资源利用率高，吞吐量大；
    • 多道
    • 宏观上并行
    • 微观上串行
  5. 分时操作系统：有交互性
  6. 及时性和可靠性强；
graph LR
F[实时操作系统] --> G[网络操作系统]
G[网络操作系统] --> H[分布式操作系统]
H[分布式操作系统] --> I[个人计算机操作系统]

操作系统的运行环境

处理器运行模式

  • OS内核(kernel)程序
  • 用户程序

前者是后者的管理者

  • 特权指令：不允许用户直接使用的指令；
    • I/O
    • 关中断
    • 内存清零
    • 存取用于内存保护的寄存器
    • 修改PSW(程序状态字寄存器)
  • 非特权指令：允许用户直接使用的指令；

OS kernel

  • 时钟管理
  • 中断机制
  • 原语
    • 处于OS的最底层，最接近硬件
    • 具有原子性，只能一气呵成
    • 程序运行时间短，调用频繁
  • 系统控制的数据结构及处理
    • 进程管理
    • 存储器管理
    • 设备管理
